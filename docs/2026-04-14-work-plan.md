## 今天要做的任务（建议新拆 722）

- 722：CKShare Scene 入口与接受链路加固 v1

交付物（最小）：

- UIKit lifecycle：`AppDelegate + SceneDelegate` 承载 SwiftUI root，确保拿到 `scene(_:willConnectTo:options:)` 与 `windowScene(_:userDidAcceptCloudKitShareWith:)` 回调。
- Scene 入口：
  - 冷启动：读取 `connectionOptions.cloudKitShareMetadata` 并触发接受流程
  - 运行/挂起态：实现 `windowScene(_:userDidAcceptCloudKitShareWith:)` 并触发接受流程
  - URL 路径：处理 `scene(_:openURLContexts:)` 作为 fallback（以及手动输入）
- 接受策略：
  - metadata 路径：直接 `CKAcceptSharesOperation`（通过 client/service 封装），不再丢弃系统 metadata
  - URL 路径：仍支持（fetch metadata -> accept），用于无 metadata 的场景与手动诊断
- 校验与降级：
  - URL 校验：scheme/host + path 形状（`/share/<token>`）本地判定，输出稳定错误码
  - availability 语义：把 transient 状态映射为 `.failed(...)`，避免误判为未授权
